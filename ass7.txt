### Q1(a). Display the name of employee who earns maximum salary.

```sql
SELECT ENAME
FROM EMP_39
WHERE SAL = (SELECT MAX(SAL) FROM EMP_39);
```

### Q1(b). Display the name of employee who earns maximum salary whose job is salesman.

```sql
SELECT ENAME
FROM EMP_39
WHERE JOB='SALESMAN'
AND SAL = (
    SELECT MAX(SAL)
    FROM EMP_39
    WHERE JOB='SALESMAN'
);
```

### Q1(c). Display the name of employee who earns minimum salary and whose job is clerk.

```sql
SELECT ENAME
FROM EMP_39
WHERE JOB='CLERK'
AND SAL = (
    SELECT MIN(SAL)
    FROM EMP_39
    WHERE JOB='CLERK'
);
```

### Q1(d). Display the department whose average salary is maximum.

```sql
SELECT DEPT_NAME
FROM
(
    SELECT DEPT_NAME, AVG(SAL) AVG_SAL
    FROM EMP_39
    GROUP BY DEPT_NAME
    ORDER BY AVG_SAL DESC
)
WHERE ROWNUM=1;
```

### Q1(h). Display the name of the department with the maximum total salary.

```sql
SELECT DEPT_NAME
FROM
(

    FROM EMP_39
    GROUP BY DEPT_NAME
    ORDER BY TOTAL_SALARY DESC
```


### Q1(j). Display the name of the city in which the manager works.

**(Your query was incorrect because it returns all managers' cities. The question asks for the city where the President/Manager's manager works.)**

```sql
SELECT CITY
FROM EMP_39
WHERE EMPNO =
(
    SELECT MGR
    FROM EMP_39
    WHERE JOB='MANAGER'
    AND ROWNUM=1

```

### Q1(k). Display the grade of the employee named 'MARTIN'.

```sql
SELECT GRADE
FROM GRADE_39
WHERE
(
    SELECT SAL
    FROM EMP_39
    WHERE ENAME='MARTIN'
)
BETWEEN MINSAL AND MAXSAL;
```

### Q1(l). List the employees who earn more than every employee in 'DALLAS'.

**(Use ALL instead of MAX for a multiple-row subquery.)**

```sql
SELECT ENAME
FROM EMP_39
WHERE SAL > ALL
(
    SELECT SAL
    FROM EMP_39
    WHERE CITY='DALLAS'
);
```

### Q1(m). Display the name of the department which has no employees.

**(Your query can never return a result because departments without employees are not stored separately.)**

```sql
SELECT DISTINCT DEPT_NAME
FROM EMP_39
MINUS
SELECT DEPT_NAME
FROM EMP_39;
```

> **Note:** With your schema, this query returns **no rows** because there is no separate Department table.

